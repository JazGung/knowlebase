# 管理后台UI设计原则
页面分为左右两栏，左边是导航栏，右边是内容区。导航栏以树状菜单形式展示当前用户有权限的页面，内容区根据导航栏的选择展示对应的功能页面。

界面根据展现方式分为两类：
1. 页面：需要在左侧导航栏尝试的项，以一个独立的html形式存在
2. 弹出框：在页面中点击各个按钮弹出的界面

界面根据数据组织方式分为以下几类：
1. 列表：从上到下分为工具栏、搜索栏和数据区3个部分：
  - 工具栏：放置操作按钮，如新建、修改等
  - 搜索栏：放置搜索输入框和搜索按钮，输入框可以是文本输入框、下拉选择框、日期选择框等，根据数据类型和业务需求选择合适的表单项类型
  - 数据区：以表格形式展示数据，表格支持分页、排序和选择功能
2. 表单：用于新建、修改、详情查看等功能，以表单形式展示数据，支持输入和修改功能，表单项可以是文本输入框、下拉选择框、日期选择框等，根据数据类型和业务需求选择合适的表单项类型；同一表单用于新建、修改、详情查看时的区别在于：新建时所有字段都可以输入，修改时部分字段不可修改，详情查看时所有字段都不可修改

工具栏的按钮按照处理方式分为两类：
1. 页面跳转：点击后打开弹出框，如新建、修改
2. 直接操作：点击后直接执行操作，如启用、停用、导出等，此时需要通过弹出框确认后才能执行操作，避免误操作

页面跳转类型的按钮可以打开一个表单；也可以打开一个列表，例如在应用管理中有个版本管理的功能，点击后打开一个版本列表的页面对选中的应用进行版本管理，此时这个版本列表页面就属于弹出框中的列表类型。  
尤其要注意工具栏按钮是否需要在数据区选中一条记录才能操作，如果需要，则要在未选中记录时将其置灰，当鼠标悬停时显示提示信息“请先选择一条记录”，以提升用户体验并避免误操作；一般情况下，只有新建按钮不需要选中记录就可以操作，其他如修改、启用、停用、导出等操作都需要选中记录才能操作。

如果系统有权限控制，则左侧导航栏的树状菜单需要根据当前用户的权限进行动态展示，只有用户没有权限的菜单也要显示，但置灰不提供点击功能，并且当鼠标悬停时显示提示信息“您没有权限访问此页面”，以提升用户体验并明确权限限制；同理界面上的按钮也需要根据权限进行动态展示，用户没有权限的按钮要置灰并提供提示信息。
