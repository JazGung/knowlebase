# 讨论规定
当对话内容属于讨论而非执行时，需要遵守以下规定：
1. 不能直接开始生成代码
2. 如果用户提出了初步设想，你需要基于设想进行细化，并对设想中的遗漏向用户进行提问，并根据用户的回答进行补充或者忽略
2. 讨论的内容如有不清晰的地方需要先进行澄清
3. 澄清时除了你给的选项之外，还必须要有“其他”和“暂不考虑”两个选项：当用户选择“其他”时需要让用户自己输入澄清内容；当用户选择“暂不考虑”时需要记录下来并在后续的计划中进行考虑，例如讨论web管理界面需求时，你需要知道如何进行操作鉴权，那用户选择“暂不考虑”说明现在不需要做鉴权相关功能
4. 如果用户要求先生成部分代码，必须遵从指令，例如用户要求先完成主体功能而不考虑如何进行对外暴露（例如通过SpringMVC或FastAPI发布为REST API），那就先完成主体功能的代码，等用户确认后再进行对外暴露的代码编写（即先完成MVC架构中的业务层，控制层后续补充）
5. 讨论的结果需根据md-file-writing-rules中的规定写入CLAUDE.md或/和README.md中
6. 如果对话模式处于plan mode时，完成上述步骤即可，后续是否需要执行根据用户明确指令进行；处于defualt mode或者accept edit on mode时，在完成上述步骤后需要想用户确认是否执行，用户明确确认后再开始生成代码，否则不执行（注意，即使处于accept edit on mode时也不能自动开始生成代码）
7. 当用户提出忽略历史讨论结果，重新开始讨论时，先读取已有结论并总结给用户，然后根据用户的输入重新生成结论，并将新的结论写入CLAUDE.md或/和README.md中
