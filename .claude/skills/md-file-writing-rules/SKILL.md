# md文件内容划分原则
- **README.md**：主要用于项目的整体介绍、技术栈、功能规划、开发步骤等内容，面向开发者和用户，提供项目的全局视图和使用指南。
- **CLAUDE.md**：主要用于记录具体的技术实现细节、设计决策和代码注释等，为开发者提供详细的技术参考。

# README.md文件内容结构
README.md的内容结构可以按照以下章节划分：
```markdown
# 项目介绍
简要介绍项目的背景、目标和核心功能。
# 功能模块
详细描述项目的功能模块划分和所使用的技术栈，这里的模块指CLAUDE.md中功能模块章节中的一级模块
# 中间件选型
中间件特指需要独立部署的组件，如数据库、消息队列等，说明选型理由和使用方式；如果是集成到编码中的库，则称为“组件（component）”则不属于中间件
# 部署指南

```

# CLAUDE.md文档结构
需按照以下章节结构编写CLAUDE.md：
```markdown
# 需求分析
## xxx
1. 步骤1
2. 步骤2
3. 步骤3
……
## yyy
### zzz
1. 步骤1
2. 步骤2
3. 步骤3
……
# 设计决策
## xxx模块设计决策
## yyy模块设计决策
### zzz子模块设计决策
```
功能模块最高一级是模块，对应README.md中的功能模块中的列举项，例如上面示例中的xxx和yyy，；非一级都称为功能点，例如上面示例中的zzz，功能点也可以按照逻辑归属关系进行嵌套，当最小的模块一定要有明确的步骤说明，且步骤内容要具体、清晰，便于开发者理解和实施。如果是简单模块，可以不在区分功能点，复杂模块则需区分；如果功能点之间在操作上有先后关系，则需在模块中明确，并以业务流程图的形式描述。
需求分析章节描述功能实现的具体步骤。
设计决策章节描述包括当不限于实现组件的选型等决策，以及为什么要这么设计、这样设计的优缺点是什么、是否有其他备选方案等。
